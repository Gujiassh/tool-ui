Shopping experiences need a clear receipt before the user commits. OrderSummary renders line items with images, quantities, and an itemized price breakdown. Pass `choice` after confirmation to switch it into a read-only receipt.

## Getting Started

<Steps>

<Step title="Install the component">

Run this once from your project root.

```bash
npx shadcn@latest add @tool-ui/order-summary
```

</Step>

<Step title="Use it in your application">

Render `OrderSummary` in your UI with tool-compatible props.

<div className="[&_pre]:max-h-[30rem] [&_pre]:overflow-y-auto">

```tsx
import { OrderSummary } from "@/components/tool-ui/order-summary";

export function Example() {
  return (
    <OrderSummary
      id="order-summary-example"
      items={[
        {
          id: "item-1",
          name: "Wireless Earbuds Pro",
          description: "Active noise cancellation",
          imageUrl: "https://example.com/earbuds.jpg",
          quantity: 1,
          unitPrice: 149.99,
        },
        {
          id: "item-2",
          name: "USB-C Charging Cable",
          description: "2m braided",
          quantity: 2,
          unitPrice: 19.99,
        },
      ]}
      pricing={{
        subtotal: 189.97,
        tax: 16.62,
        shipping: 0,
        total: 206.59,
        currency: "USD",
      }}
    />
  );
}
```

<Step title="Register it in your runtime provider">

Register this renderer so tool results display as `OrderSummary`.

```tsx
"use client";

import { type Toolkit } from "@assistant-ui/react";
import { OrderSummary } from "@/components/tool-ui/order-summary";
import {
  safeParseSerializableOrderSummary,
  SerializableOrderSummarySchema,
} from "@/components/tool-ui/order-summary/schema";
import {
  ToolUI,
  createArgsToolRenderer,
  createDecisionResult,
} from "@/components/tool-ui/shared";

export const toolkit: Toolkit = {
  createOrder: {
    description: "Display a purchase order for user confirmation",
    parameters: SerializableOrderSummarySchema,
    render: createArgsToolRenderer({
      safeParse: safeParseSerializableOrderSummary,
      idPrefix: "order",
      render: (parsedArgs, { result, addResult }) =>
        result ? <OrderSummary {...parsedArgs} choice={result} /> : (
          <ToolUI id={parsedArgs.id}>
            <ToolUI.Surface>
              <OrderSummary {...parsedArgs} />
            </ToolUI.Surface>
            <ToolUI.Actions>
              <ToolUI.DecisionActions
                actions={[
                  { id: "cancel", label: "Cancel", variant: "outline" },
                  { id: "confirm", label: "Purchase", variant: "default" },
                ]}
                onAction={(action) =>
                  createDecisionResult({
                    decisionId: `${parsedArgs.id}-decision`,
                    action,
                    payload:
                      action.id === "confirm"
                        ? {
                            orderId: `ORD-${Date.now()}`,
                            confirmedAt: new Date().toISOString(),
                          }
                        : undefined,
                  })
                }
                onCommit={async (decision) => {
                  if (decision.actionId !== "confirm") return;
                  const orderId =
                    typeof decision.payload?.orderId === "string"
                      ? decision.payload.orderId
                      : `ORD-${Date.now()}`;
                  const confirmedAt =
                    typeof decision.payload?.confirmedAt === "string"
                      ? decision.payload.confirmedAt
                      : new Date().toISOString();
                  await addResult?.({
                    action: "confirm",
                    orderId,
                    confirmedAt,
                  });
                }}
              />
            </ToolUI.Actions>
          </ToolUI>
        ),
    }),
  },
};
```

</Step>

</Steps>

## Key Features

<FeatureGrid>
  <Feature icon="DollarSign" title="Currency formatting">
    Locale-aware prices via `Intl.NumberFormat` for any ISO currency code
  </Feature>
  <Feature icon="List" title="Pricing breakdown">
    Subtotal, tax, shipping, discounts, and total in one place
  </Feature>
  <Feature icon="CheckCircle2" title="Receipt state">
    Dims the card and shows order ID after purchase confirmation
  </Feature>
  <Feature icon="Package" title="Image fallbacks">
    Package icon placeholder when a product image is missing
  </Feature>
</FeatureGrid>

## Receipt State

Pass `choice` to render a dimmed receipt with the order ID and timestamp.


## With Discount

Add `discount` and `discountLabel` to `pricing` to show promotional savings.

## Accessibility

- Uses semantic `<article>` with `aria-labelledby` pointing to the title
- Action buttons are keyboard accessible with visible focus states
- Images use `alt=""` since item names provide the description
- Receipt state shows clear visual indication of confirmation
